## 🎯 Step-by-Step Procedure

### Phase 1: Font Setup (10 minutes)

#### Step 1.1: Download Font
```bash
# Visit https://opendyslexic.org/
# Download and extract
# Place OpenDyslexic-Regular.otf in chrome-extension/fonts/
```

#### Step 1.2: Encode Font
```bash
cd chrome-extension
node embed-font.js
```

**Output:** Creates `opendyslexic-base64.css` with base64 font

#### Step 1.3: Update contentScript.js
1. Open `contentScript.js`
2. Find `injectDyslexiaStyles()` function (line ~100)
3. Copy `@font-face` CSS from `opendyslexic-base64.css`
4. Paste at the beginning of `styleEl.textContent`

**Before:**
```javascript
styleEl.textContent = `
  body.dyslexia-mode-active * {
    font-family: "OpenDyslexic", Arial, Verdana, sans-serif !important;
```

**After:**
```javascript
styleEl.textContent = `
  @font-face {
    font-family: 'OpenDyslexic';
    src: url('data:font/otf;base64,YOUR_BASE64_STRING_HERE') format('opentype');
    font-weight: normal;
    font-style: normal;
  }
  
  body.dyslexia-mode-active * {
    font-family: "OpenDyslexic", Arial, Verdana, sans-serif !important;
```

If you want to integrate with the AdaptEd webapp:

### Frontend Integration

1. Copy `bridge.js` to `frontend/src/lib/dyslexia-bridge.js`

2. Use in citation link component:
```typescript
import { enableDyslexiaMode, isDyslexiaModeAvailable } from '@/lib/dyslexia-bridge'

  // Enable dyslexia mode if extension is installed
  if (isDyslexiaModeAvailable()) {
    enableDyslexiaMode()
  }
  
  // Open link
  window.open(url, '_blank')
}
```

## 🐛 Common Issues & Solutions

### Issue 1: Font not changing
**Cause:** Font not embedded or incorrect base64  
**Solution:** Re-run `embed-font.js` and verify `@font-face` in contentScript.js

### Issue 2: Icons not showing
**Cause:** Icon files missing or wrong names  
**Solution:** Generate icons using `create-icons.html` or `generate-icons.js`

### Issue 3: Popup not opening
**Cause:** popup.html missing or manifest incorrect  
**Solution:** Verify all popup files exist and manifest has `action.default_popup`

### Issue 4: Global mode not working
**Cause:** Message listener not set up  
**Solution:** Verify contentScript.js has `chrome.runtime.onMessage` listener
